LEAD

Latent Entropy-Aware Decoding for Reliable Multimodal Reasoning

Thinking in Uncertainty: Mitigating Hallucinations in MLRMs with Latent Entropy-Aware Decoding. LEAD 是一个即插即用的多模态推理解码策略：当模型进入高熵、不确定的推理阶段时，保留 token 分布中的潜在语义；当熵下降时，再切回离散 decoding 完成稳定收敛，并通过视觉锚点把推理重新拉回图像证据。

🔍
关键发现：转折词和过渡词经常对应高熵推理状态，而幻觉往往在这些节点之后更容易出现。
🧠
熵感知切换：高熵阶段采用概率加权的连续 embedding 进行 latent reasoning，低熵阶段恢复离散 token decoding 以保证收敛性。
🖼️
视觉锚点注入：在高不确定推理的关键时刻注入视觉 anchor，减少脱离图像内容的“脑补式”推理。
📈
实验收益：LEAD 在幻觉评测、通用理解、数学与科学推理上都带来稳定增益，同时兼顾更短 reasoning length 与更高文本质量。

方法概览

LEAD 的核心思想是把 token-level entropy 当作推理不确定性的信号。当模型处于高熵状态时，单个采样 token 往往不足以表达真实的语义竞争关系，因此 LEAD 直接利用完整 token 分布构造概率加权 embedding，保留多个候选语义；当熵回落后，再切换回标准离散 decoding。与此同时，方法还会在关键的不确定阶段注入视觉锚点，强化视觉 grounding，减少幻觉传播。

LEAD 在高熵阶段执行 latent decoding，在低熵阶段恢复 discrete decoding；视觉与文本 token 在统一的多模态推理框架中共同驱动生成。

关键观察与动机

这篇工作首先从推理过程本身出发，分析幻觉出现的位置与不确定性的关系。一个直接发现是：在多模态大推理模型中，幻觉往往与 because、however、so、but 等转折或过渡节点共现。进一步的 token-level 分析显示，高熵 token 并不是可以忽略的噪声，它们反而是决定后续推理方向的关键节点；并且当这些高熵 token 与幻觉相关时，模型通常分配给视觉内容的注意力更少。

幻觉经常在 transition words 附近出现，而且这些案例占总体 hallucination 的显著比例，说明高不确定推理节点与错误推断密切相关。
高熵 token 对最终性能影响更大，越早出现的高熵 token 越会影响整条 reasoning trajectory；与 hallucination 相关的高熵 token 通常具有更低的视觉注意力。

定性可视化分析

LEAD 不仅改善最终答案，还改变了模型在推理过程中的行为模式。从可视化上看，方法能够在高熵阶段维持更丰富的 token 分布，并在关键时刻重新分配视觉注意力，从而避免模型在不确定阶段脱离图像内容继续“顺着语言惯性往下想”。

LEAD 在 reasoning steps 中保持更稳定的视觉关注，同时在 latent reasoning 阶段保留更分散的 token 概率分布，体现出“探索—收敛”两阶段的动态切换。

主要结果

实验结果表明，LEAD 不只是提升单一基准上的最终分数，而是在 sample efficiency、reasoning efficiency 与文本质量之间取得了更好的综合平衡。它能在更小的 k 下更快达到更高的 Pass@k 准确率，也能以更短的平均推理长度获得更高的正确率，并在 fluency、naturalness、grammar 与 perplexity 上维持或提升生成质量。

在 RealWorldQA 与 MathVista 上，LEAD 的 Pass@k 曲线整体高于对比方法，并更早达到高精度区间，体现出更强的 sample efficiency。
LEAD 以更短的平均 reasoning length 获得更高准确率，说明 latent reasoning 并没有拉长推理，反而提高了思考效率。
在 GPT-assisted evaluation 中，LEAD 在 fluency、naturalness、grammar 以及 PPL 指标上保持稳定甚至更优，说明方法不会以牺牲文本质量为代价换取更高准确率。

消融研究

消融结果进一步验证了 LEAD 的设计选择。动态 entropy threshold 优于固定阈值策略，说明推理过程中需要根据不确定性自适应切换 reasoning mode；此外，discrete reasoning window 也并非越大越好，中等大小的 window 更能在语义探索与稳定收敛之间取得平衡。

在 MMHalu 和 Bingo 上，动态阈值策略（Δ） consistently 优于固定阈值，说明自适应切换是 LEAD 成功的关键因素之一。
Window size 提升到中等范围时性能最好，过小会影响稳定性，过大则会削弱 latent reasoning 带来的探索优势。
